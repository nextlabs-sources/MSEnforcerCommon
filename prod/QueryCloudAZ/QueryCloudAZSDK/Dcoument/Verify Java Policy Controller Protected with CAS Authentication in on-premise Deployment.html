Verify Java Policy Controller Protected with CAS Authentication in on-premise Deployment
Duan Shiqiang

1. Summary

After setting up the deployment, one can use Rest Client like Chrome's plugin Postman to verify whether it's working or not.

In this documentation, the hostname/FQDN of the Java Policy Controller will be referred as {JPC_HOST}.

2. Procedure

2.1. Install Postman

You can install the plugin at Postman's Chrome Stroe Page.

2.2. Access the REST APIs using Postman

Note: Before you hit the REST APIs using Postman, you should access the REST API endpoint once directly using your Chrome and acknowlege the Chrome to accept the JPC server's self-signed certificate.

  1. Getting Ticket Granting Ticket

The first step of the authentication process is to get the Ticket Granting Ticket (TGT), in Postman, you can invoke a POST call to url: https://​{JPC_​HOST}/​cas/​v1/​tickets with x-www-form-urlencoded data:

  • username: ${Your Control Center username}
  • password: ${Your Control Center password}

A screenshot for the request:

After you get the response, you can switch to be reponse header section and copy the Location header value:

  1. Getting Service Ticket

The second step of the authentication process is to get the Service Ticket (ST). The url to request is the Location header value get from 1st step. In Postman, you can invoke a POST call to the url with x-www-form-urlencoded data:

  • service: https://{JPC_HOST}/dpc/authorization/pdp

A screenshot for the request:

After you get the response, copy the response text which is the ST.

Beware, the ST expires in 20 seconds, and it's used in next step, so you must conduct the next step very quickly. (You can save the request as template in Postman, doing so will be giving enough time).

  1. Getting Authentication Cookie

The 3rd step of the authentication process is to get the Authentication cookie using ST. In postman, you can invoke a GET call to the url: https://​{JPC_​HOST}/​dpc/​authorization/​pdp with query string parameter:

  • ticket: The ST value

A screenshot for the request:

  1. Make PDP REST API Evaluation Call

With 3rd step completed, you can now make PDP REST API calls from Postman. In postman, you can invoke a POST call to url https://​{JPC_​HOST}/​dpc/​authorization/​pdp with x-www-form-urlencoded data:

  • Version: 1.0
  • DataType: json or xml
  • Service: EVAL
  • data: ${the request payload data, should use json version if you specify json as DataType or xml if you specify xml as DataType}

The authentication cookie expires in about 20 mins, after that you need redo the whole process to make PDP REST call.
